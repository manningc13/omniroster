# Cogitator

A Warhammer 40,000 list builder for Adeptus Mechanicus with a dynamic rules engine — stat cards update live based on Doctrina Imperatives, stratagems, and other per-turn choices.

## Stack

| Layer | Technology |
|---|---|
| Frontend | Vite + React + TypeScript + Zustand |
| Rules engine | Rust → WebAssembly (wasm-pack) |
| Unit data | Community JSON (Wahapedia) |
| Hosting | VPS (nginx) |
| CI/CD | GitHub Actions |

## Local setup

### Prerequisites

```bash
# Rust toolchain
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
rustup target add wasm32-unknown-unknown

# wasm-pack
cargo install wasm-pack
```

### First run

```bash
# From repo root — builds WASM engine then starts frontend dev server
npm run dev
```

The frontend has a JavaScript fallback engine so it's usable immediately even before the WASM build completes. The WASM build replaces it automatically.

### Run engine tests

```bash
cd engine && cargo test
```

## Adding a new modifier

1. Add the variant to `Modifier` enum in `engine/src/types.rs`
2. Add a match arm in `apply_modifier()` in `engine/src/resolver.rs`
3. Write tests in the same file
4. Mirror the type in `app/src/types/index.ts`
5. Add the JS fallback case in `app/src/engine/index.ts`

## VPS deployment

### One-time server setup

```bash
# On your VPS (Ubuntu)
sudo apt install nginx
sudo mkdir -p /var/www/cogitator
sudo chown $USER:$USER /var/www/cogitator
sudo cp nginx.conf /etc/nginx/sites-available/cogitator
sudo ln -s /etc/nginx/sites-available/cogitator /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

### GitHub Actions secrets required

| Secret | Value |
|---|---|
| `VPS_HOST` | Your VPS IP or domain |
| `VPS_USER` | SSH user (e.g. `ubuntu`) |
| `VPS_SSH_KEY` | Private SSH key with access to the VPS |

Push to `main` to trigger a deploy. PRs run tests only.
